Clock Drift Compensation/Calibration– Mk II Non-Autosync Units Only

The clocks in the receiver and transmitter need to be synced in order to measure the distance between the two units. Time-of-flight is used for this. Slight differences in crystal manufacturing can cause the two clocks to run at slightly different speeds. Clock drift compensation is used to compensate for some of the difference. Autosync units do this each time satellite lock is achieved. For units without autosync the method is as follows:
  • Put transmitter and receiver at a rigidly fixed distance from each other and at the same temperature (example, taped together in a 5-gallon bucket of water, or even just taped together on the bench)
  • Power up transmitter
  • Power up receiver and make sure it has acquired the transmitter.
  • If you want progress updates on the calibration process so you don’t have to stare nervously at the setup hoping it’s actually doing something, send a “Verbose 2” (“V2”) command.
  • Send receiver a “Crystal Calibration” command (the “E” command if you are sending free text commands or click the “Begin Crystal Calibration” button if you are using the CeruleanTracker App)
  • Wait 30 minutes without disturbing positions or sending other commands. The receiver will send an informational message when complete
This calibration is to be done when first put into service to account for crystal manufacturing tolerances and then about every three months to account for crystal aging in both the transmitter and receiver. Aging is proportional to the time the units are powered, and the most aging occurs in the first few days of being powered. Crystal manufacturing tolerances normally have a much larger effect than aging.
