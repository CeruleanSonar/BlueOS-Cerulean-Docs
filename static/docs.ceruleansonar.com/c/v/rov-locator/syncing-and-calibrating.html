Syncing and Calibrating

There are two main types of user calibrations:
  • Clock drift compensation (should be done when put into service and then every three months or so) – only for Mk II non-autosync units (autosync units do this automatically, and Mk III units use a different methodology that does not require clock calibration)
  • IMU Calibration
There are three methods of syncing the Mk II transmitter to the receiver:
  • Fast sync with dumb or autonomous transmitter– only for non-autosync units
  • Commanded sync with tethered transmitter and receiver– only for non-autosync units
  • Autosync units do this automatically
  • Mk III units don’t need to do this
The clock drift calibration is used to match the crystals in the two units so the timebases run at the same speed. Since the crystals age at slightly different rates, this needs to be repeated periodically, and we recommend every three months. This calibration is stored in non-volatile memory on the receiver.
The fast sync (“Dx” command) or commanded sync (“A!” command) need to be done every time power is interrupted to either the receiver or transmitter – only for non-autosync units. Since the systems start at different times, the timebases are in slightly different phases in the processing cycles. Think of it like the clock on your microwave. When the power is interrupted, the microwave flashes “12:00” until someone sets it. The ROVL commanded sync (“A!” command) sets both timebases to the same “time” referenced to the receipt of your external command. Alternatively, the fast sync (“Dx” command) sets just the receiver to the “time” the transmitter is using, which is technically a different thing but has the same effect. For the Dx command, since the receiver and transmitter are linked acoustically, the time of flight of the transmitter ping needs to be taken into account – this is where the known distance comes into play. You can have the units any distance apart (including touching) when you send the “D” command. Or, you can eyeball it – you see the ROV is in the water and it’s two or three meters from the receiver, so send a “D2.5” command. It’s going to be a little off, but chances are very good you will never notice the error. The minimum distance accepted is 0.1 meters.
