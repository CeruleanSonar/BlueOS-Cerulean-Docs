Scenario: Holding Position

When flying aerial drones, it is typical to be able to release the controls and have the drone hold its position. For ROVs, since we normally don’t have an external position reference equivalent to the drone’s GPS, when we release the controls, the ROV usually drifts due to currents and pull from the tether.
Using the DVL, we can lock into a position and hold it while the operator is doing other tasks.
What this requires in addition to the DVL: For this to work, the ROV control software must have a working position hold function, and it also must be able to accept data from the DVL. For example, the BlueROV Companion software listens on Ethernet UDP port 27000 for a GPS $GPRMC message. The Cerulean DVL can emulate a GPS sending unicast or broadcast $GPRMC messages to port 27000 (or another port of your choice), and it also sends the same $GPRMC message to the serial interface. The DVL also outputs a $DVPDL message with position and orientation data in ROV coordinates, which can be used by other’s software for position holding. Note: as of this writing, the Blue Robotics software suite lists an option for position hold, but it is not implemented. Cerulean can supply an integration software that connects the DVL to the PixHawk in a BlueROV, implementing position hold. Contact us for more information.
