Device Discovery and IP Addressing

When SonarView starts it automatically initiates the device discovery process and shows any devices found. In this example it has found the S500 Sounder.
SonarView and Cerulean sonars with Ethernet interfaces and Ping360 on Ethernet interface fully support the device discovery protocol. This protocol includes the ability to update the device's IP address.
​
The discovery protocol will identify any devices attached to the network regardless of their current IP address. If the Connect button is disabled, it is because the device's IP address is not in the subnet of any of the current Ethernet connections, and so it cannot be addressed currently.
You have the opportunity to change the device's IP address from this screen by clicking
and entering the desired IP address then clicking Enter.
If you need to change the IP address of a device, make sure there is only one device of that type on the subnet. The Set IP function uses a broadcast message so if there are multiple devices, they will all change to the new IP address!
If the IP address is in the subnet of your Ethernet connections, you can click "Connect" to open socket connection and activate the appropriate viewer module.
SonarView will also find S500 and Ping sounders on serial interfaces, and any devices routed via BlueOS through the tether on ports 9090 and 9092.
